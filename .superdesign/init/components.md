# Shared UI Components

The application uses vanilla HTML and TypeScript rather than a component framework. There is no
`src/components/` directory and no third-party component library. Shared visual primitives are
defined as semantic CSS classes in `src/styles/app.css` and instantiated in `index.html` or in the
feature render functions.

## DOM helpers

Source: `src/shared/dom.ts`

These are the only shared UI-adjacent helpers. They provide typed required-element lookup and
multi-element lookup.

```ts
export function getElement<T extends Element>(selector: string): T {
  const element = document.querySelector<T>(selector);
  if (!element) throw new Error(`Missing required element: ${selector}`);
  return element;
}

export function getElements<T extends Element>(selector: string): T[] {
  return [...document.querySelectorAll<T>(selector)];
}
```

## CSS primitives

The reusable primitives are `.button` and its variants, `.card`, `.field`, `.input`, `.select`,
`.textarea`, `.toggle`, `.icon-button`, `.status-dot`, `.sync-badge`, `.section-head`, `.modal`,
`.toast`, and responsive grid classes. Their complete implementation is captured in `theme.md`.

## Feature renderers

Page-specific visual components are generated by these complete source files and must be passed as
context when designing their corresponding view:

- `src/features/dashboard/index.ts`: stat content, tracker cards, activity rows, and seven-day chart.
- `src/features/history/index.ts`: grouped history rows, summaries, and filters.
- `src/features/trackers/index.ts`: tracker management cards and tracker modal state.
- `src/features/logs/index.ts`: log modal state and record editing interactions.
- `src/features/settings/index.ts`: settings content and account/sync state.
- `src/features/auth/index.ts`: signed-in/signed-out presentation and auth form state.
